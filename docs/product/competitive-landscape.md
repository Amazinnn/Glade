# Glade Competitive Landscape

Last updated: 2026-05-01

This document is a positioning tool, not a product catalog.

It maps representative products across adjacent categories so Glade can make clearer choices about:

- what it is
- what it is not
- what it should borrow
- what it should refuse

Locked positioning premises:

- **Primary category:** local-first life system
- **Primary wedge:** life-first
- **Primary audience:** busy but sensitive people
- **Core loop:** record -> recover -> gently reflect

This map is intentionally representative rather than exhaustive.

## Positioning Synthesis

### What Glade does not directly compete on

Glade should not try to win on:

- enterprise or team collaboration
- unlimited database flexibility
- template ecosystems
- AI writing workstation features
- generic productivity control centers
- broad wellness content libraries

This means Glade is **not**:

- a lighter Notion
- a prettier AI journal
- a bundle of focus, meditation, habit, and note apps under one roof

### What Glade should borrow

Glade should selectively borrow:

- from **Notion**: multi-view thinking, not database-first life
- from **Obsidian**: file ownership and long-term trust
- from **Evernote**: search confidence and retrieval speed
- from **Day One**: memory value and rereadability
- from **Things / Todoist / Sunsama**: gentle planning rhythm and low-friction capture
- from **Daylio / Oura / Apple Health**: state lenses and long-term pattern awareness
- from **OmmWriter / iA / Tide**: calm, minimal, thought-supporting states
- from **Finch / Plant Nanny / Forest**: visible growth through care, not pressure

### Glade's primary differentiating combination

Glade's strongest combination is:

- **life-first**
- **local-first**
- **capture first**
- **Agent-forward but raw-grounded**
- **structure grows from life flow**
- **garden, not workspace**

In practice, that means:

- the user gives Glade raw material first
- the material lands in trusted local records
- Agent helps shape material without hiding the source
- the past is easy to recover
- light structure emerges afterward
- reflection stays gentle

### Future abilities Glade can absorb without losing itself

Future-compatible abilities include:

- Zen mode
- meditation mode
- focus mode
- mood and health lenses
- flower-growth feedback
- careful agent support

These only fit if they continue serving the same loop:

**record -> recover -> gently reflect**

They should not become separate product lines inside the app.

## Final Positioning Conclusions

### Primary category

Glade should be defined as a **local-first life system**.

### Primary wedge

Glade should be remembered first for being **life-first**, not for AI, tasks, or databases.

### Core emotional difference

Glade is a **garden**, not a workspace.

### Near-term strategic center

Glade should first become excellent at:

- recording
- recovering
- light understanding

It should not lead with:

- platformized agents
- complex views
- heavy task systems

### Modularity lesson

Glade should borrow plugin power without becoming a plugin-maintenance hobby.

The right lesson from tools like Obsidian, Joplin, and agent runtimes is not "make everything a marketplace." The useful lesson is that small capabilities should be bounded, switchable, and replaceable. In Glade, plugins should serve ability modules and interface orchestration surfaces without making the user manage the system before recording life.
